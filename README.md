# CS-320---Software-Test-Automation-QA
## How can I ensure that my code, program, or software is functional and secure?

I ensure that my software is functional and secure by following software testing best practices throughout the development process. I create unit tests to verify that each class and method meets the project requirements, including normal operations, boundary conditions, and invalid inputs. I also review my code for correctness, follow secure coding practices, and make changes only after verifying that existing functionality still works. Running automated JUnit tests throughout development helped me identify defects early and improve the overall reliability of the application.

## How do I interpret user needs and incorporate them into a program?

I begin by carefully reviewing the software requirements and identifying the functionality the customer expects. For this project, I translated the requirements into Java classes, service classes, and unit tests that enforced each business rule. I focused on validating user input, preventing invalid data from being stored, and ensuring that the application behaved exactly as specified. Breaking the requirements into smaller tasks helped me build the application in a structured and organized manner.

## How do I approach designing software?
I approach software design by first understanding the problem before writing code. I divide the application into smaller, reusable components that each have a single responsibility. I then design classes that are easy to maintain, test, and extend. Throughout this course, I learned that combining object-oriented design principles with comprehensive unit testing produces software that is reliable, maintainable, and easier to improve as requirements evolve.
